API Endpoints

POST /solve

Solve a MaxCut optimization problem for the given graph.

Headers

Name Type Description
X-API-Keyrequired string Your API authentication key
Content-Typerequired string Must be application/json

Request Body

Field Type Description
num_nodesrequired integer Number of nodes in the graph (2-200)
edgesrequired array Array of edge objects defining the graph
edges[].sourcerequired integer Source node index (0-based)
edges[].targetrequired integer Target node index (0-based)
edges[].weightrequired float Edge weight (positive value)

Response

Success response JSON 
{
  "cut_value": 42.5,
  "partition": [0, 1, 1, 0, 1, 0],
  "solve_time_ms": 23.4,
  "method_used": "GNN",
  "node_count": 6,
  "edge_count": 9
}

Response Fields

Field Type Description
cut_value float Total weight of edges crossing the partition
partition array Binary array indicating partition assignment (0 or 1) for each node
solve_time_ms float Time taken to solve in milliseconds
method_used string Algorithm used (GNN, LOCAL_SEARCH, or GUROBI)
node_count integer Number of nodes in the graph
edge_count integer Number of edges in the graph

Code Examples

Complete Python example Python 
import requests
import json

# Configuration
API_KEY = "your_api_key_here"
API_URL = "https://api.maxcut-solver.com/solve"

def solve_maxcut(nodes, edges):
    """
    Solve a MaxCut problem using the MaxCut API
    
    Args:
        nodes: Number of nodes in the graph
        edges: List of edges with source, target, and weight
    
    Returns:
        dict: Solution with partition and cut value
    """
    headers = {
        "X-API-Key": API_KEY,
        "Content-Type": "application/json"
    }
    
    payload = {
        "num_nodes": nodes,
        "edges": edges
    }
    
    try:
        response = requests.post(API_URL, headers=headers, json=payload)
        response.raise_for_status()
        return response.json()
    except requests.exceptions.RequestException as e:
        print(f"Error: {e}")
        return None

# Example: Delivery route optimization
if __name__ == "__main__":
    # Define your delivery network
    delivery_network = {
        "num_nodes": 6,
        "edges": [
            {"source": 0, "target": 1, "weight": 2.5},  # Hub to Zone A
            {"source": 0, "target": 2, "weight": 3.1},  # Hub to Zone B
            {"source": 1, "target": 3, "weight": 1.8},  # Zone A to Zone C
            {"source": 2, "target": 4, "weight": 2.2},  # Zone B to Zone D
            {"source": 3, "target": 5, "weight": 1.5},  # Zone C to Zone E
            {"source": 4, "target": 5, "weight": 2.8},  # Zone D to Zone E
        ]
    }
    
    # Solve the optimization problem
    result = solve_maxcut(
        delivery_network["num_nodes"], 
        delivery_network["edges"]
    )
    
    if result:
        print(f"Optimal Cut Value: {result['cut_value']}")
        print(f"Partition: {result['partition']}")
        print(f"Solve Time: {result['solve_time_ms']}ms")
        
        # Interpret results
        print("\nDelivery Zone Assignment:")
        for i, partition in enumerate(result['partition']):
            print(f"  Zone {i}: Route {'A' if partition == 0 else 'B'}")
Complete JavaScript example JavaScript 
const API_KEY = 'your_api_key_here';
const API_URL = 'https://api.maxcut-solver.com/solve';

async function solveMaxCut(numNodes, edges) {
    const headers = {
        'X-API-Key': API_KEY,
        'Content-Type': 'application/json'
    };
    
    const payload = {
        num_nodes: numNodes,
        edges: edges
    };
    
    try {
        const response = await fetch(API_URL, {
            method: 'POST',
            headers: headers,
            body: JSON.stringify(payload)
        });
        
        if (!response.ok) {
            throw new Error(`HTTP error! status: ${response.status}`);
        }
        
        return await response.json();
    } catch (error) {
        console.error('Error:', error);
        return null;
    }
}

// Example: E-commerce warehouse distribution
async function optimizeWarehouseRoutes() {
    // Define warehouse network
    const warehouseNetwork = {
        numNodes: 8,
        edges: [
            {source: 0, target: 1, weight: 4.2},  // Main to North
            {source: 0, target: 2, weight: 3.8},  // Main to South
            {source: 1, target: 3, weight: 2.1},  // North to NE
            {source: 1, target: 4, weight: 2.5},  // North to NW
            {source: 2, target: 5, weight: 1.9},  // South to SE
            {source: 2, target: 6, weight: 2.3},  // South to SW
            {source: 3, target: 7, weight: 1.5},  // NE to East
            {source: 6, target: 7, weight: 3.2},  // SW to East
        ]
    };
    
    const result = await solveMaxCut(
        warehouseNetwork.numNodes,
        warehouseNetwork.edges
    );
    
    if (result) {
        console.log('Optimization Results:');
        console.log(`Cut Value: ${result.cut_value}`);
        console.log(`Solve Time: ${result.solve_time_ms}ms`);
        
        // Visualize partition
        console.log('\nWarehouse Zone Assignment:');
        const zones = ['Main', 'North', 'South', 'NE', 'NW', 'SE', 'SW', 'East'];
        result.partition.forEach((group, i) => {
            console.log(`  ${zones[i]}: Distribution Group ${group + 1}`);
        });
    }
}

// Run the optimization
optimizeWarehouseRoutes();

Best Practices

1. Efficient Graph Representation

Use the minimum necessary edges to represent your network:

  • Remove duplicate edges (keep only one edge between any two nodes)
  • Use 0-based node indexing
  • Ensure all node indices are within range [0, num_nodes-1]

2. Optimal Problem Sizes

While we support up to 200 nodes, consider these guidelines:

  • 2-50 nodes: Real-time applications (<50ms response)
  • 50-100 nodes: Interactive applications (<500ms response)
  • 100-200 nodes: Batch processing (<2s response)

3. Error Handling

Always implement proper error handling:

  • Check HTTP status codes
  • Implement exponential backoff for rate limits
  • Log errors for debugging
  • Have fallback strategies for critical operations

4. Security

Keep your API key secure:

  • Never expose keys in client-side code
  • Use environment variables for key storage
  • Rotate keys periodically
  • Monitor usage for anomalies
